01. このマニュアルについて

このマニュアルは、SOLUNA が運営する 4 物件（WHITE HOUSE 熱海 / THE LODGE / NESTING / インスタントハウス）の運用に必要な情報を 1 冊にまとめたものです。 ゲスト向けの「ハウスマニュアル」とは別に、管理者と現地スタッフが使う運用手順書として作成されています。

SOLUNA admin（鍵管理 / Wi-Fi 設定 / 監査）／ 現地スタッフ（チェックイン対応 / 鍵プログラム）／ KAGI iOS app を運用する人。

03. 初回ログインとプロフィール登録

SOLUNA admin / 管理ページにはじめてアクセスする際は、必ず 名前 と 電話番号 の登録が必要です。緊急時のご連絡や監査ログのために使います。

ログイン方法は 2 通り

📧 メール OTP

  1. メールアドレスを入力
  2. 6 桁のコードがメールで届く
  3. コードを入力 → ログイン完了

💬 LINE で続行

  1. 「LINE で続行」をタップ
  2. LINE アプリで認可
  3. SOLUNA に自動でログインLINE 表示名が名前にプリフィル
💡 LINE Login が表示されない場合

LINE Login は LINE_LOGIN_CHANNEL_ID + LINE_LOGIN_CHANNEL_SECRET が設定されているときだけ画面に出ます。詳しくは「メンバー管理」章の LINE Login セットアップ手順を参照。

プロフィール入力画面

ログイン直後に https://solun.art/profile が表示されます。下記 2 項目を入力して「保存して続ける」。

05. 鍵番号のローテーション手順

チェックアウトごと、または最低でも 30 日に 1 回は鍵番号をローテーション（変更）することを推奨します。ローテーションすると旧コードは即座に無効になります。

🚨 在室ゲストへの影響

現地に滞在中のゲストがいる状態でローテートすると、そのゲストは締め出されます。
システムが自動で在室判定し、警告ダイアログを出しますが、最終確認は人間が行ってください。

標準フロー

  1. /admin/secrets を開く対象物件のカードを探す
  2. "Door code" 行の ⟳ ローテート ボタンを押す確認モーダルが開く
  3. 確認ダイアログ「鍵番号を新しい 6 桁に変更します。既存のゲストはアクセスできなくなります。続けますか？」 → OK
  4. 在室ゲストいる場合の二段階確認サーバーが 412 を返し、ブラウザは「いまご滞在中のお客様が N 組います: ...」を表示。本当に問題ない場合のみ OK
  5. 新しい鍵番号がクリップボードにコピーされるトースト「新しい鍵番号: 530955 (コピーしました)」を表示。 DB と画面が即時更新
  6. 同期バッジが 🟡 → 🟢 に変わるのを待つKAGI iOS app が SwitchBot に書き込み完了 → 自動で 🟢 になる。最大 15 分
  7. マニュアルページで動作確認/<slug>/manual を開き、Quick info カードの鍵番号が新しいものになっているか確認

非常時 (KAGI 連携が動かない場合)

  1. 同期バッジ 🟡 が 15 分以上続くKAGI iOS app の Background Refresh が止まっている可能性
  2. KAGI iOS app を起動アプリを開くだけで foreground sync が走る
  3. 「🔄 鍵を同期」ボタンを物件パネルから手動押下即時に SOLUNA → SwitchBot 書き込みが実行される
  4. それでも🔴 になる場合SwitchBot Hub のネット接続 / ロックのバッテリーを確認。最終手段として現地スタッフが手動プログラム
📌 旧コードの「監査用」保持 (7 日間)

ローテーション後、旧コードは previous_door_code カラムに 監査目的で 7 日間保存されます。 解錠ではなく「あとから誰がいつ何の番号を回したか」を追うための情報です。

❌ 物理ロックは旧コードを受け付けません — KAGI iOS が SwitchBot に書き込んだ瞬間に旧コードは無効。
✓ /admin/secrets の Last edit 行や /admin/secrets/<slug>/audit から「直前の番号は X だった」を確認できる

🔔 24h 経過アラート (Telegram watchdog)

ローテーション後 24 時間以上 KAGI iOS からの sync-ack が返らない場合、 soluna-web の watchdog が自動で Telegram (@yukihamada_ai_bot, chat 1136442501) に通知します。 KAGI iOS app が止まっている / SwitchBot が WiFi 切れ / バッテリー切れ等の検知に使ってください。

06. Wi-Fi パスワード / チェックイン時刻の変更

鍵番号と違い、Wi-Fi / 時刻はローテートではなく 編集モーダル で値を直接入力します。

編集手順

  1. /admin/secrets を開く
  2. 対象フィールドの ✎ ボタンを押す編集モーダルが開く
  3. 新しい値を入力時刻は HH:MM 形式（例: 16:00）。Wi-Fi PW は 128 文字以内
  4. 「保存」を押すトーストで保存確認 → モーダルが閉じる
  5. マニュアルページで反映確認/<slug>/manual をリロードして新しい値が出ているか

Wi-Fi 編集の注意

⚠ 物理ルーターと一致させる

DB を変えても物理ルーターの SSID/PW は変わりません。
Wi-Fi 変更時は必ず：

  1. (1) 現地ルーターの設定画面で SSID/PW を更新
  2. (2) Wi-Fi が新パスワードで実際に繋がることを確認
  3. (3) 確認後に /admin/secrets で同じ値を入力

順番を逆にすると、ゲストに「Wi-Fi 繋がらない」苦情が来ます。

チェックイン時刻の運用

  • 標準: 16:00 イン / 10:00 アウト（北海道 3 物件）、16:00 / 11:00（atami）
  • 変更例: シーズン繁忙期に 15:00 イン → 在室時間延長
  • 過去の予約への影響: Beds24 側の予約は変更されない。新規予約から適用

07. メンバー管理 — admin を増やす / LINE Login

SOLUNA の運用を複数人で行うために、admin / staff / cleaner / construction の 4 つの役割を招待型で追加できます。owner だけが招待可能です。

役割と権限

役割権限
ownermail@yukihamada.jp 固定。全権限。剥奪不可
admin/admin/secrets 編集・⟳ ローテート・メンバー招待
staff各物件マニュアル本文閲覧可（編集不可）
cleaner清掃担当向け閲覧（鍵番号も見える）
construction建設工事担当向け閲覧

新しい admin を招待

  1. https://solun.art/admin/team を開くowner / admin のみアクセス可
  2. 「NEW INVITE」フォームに招待先メール + 役割を入力
  3. 「招待を送る」を押す72 時間有効・1 回限り使用可能な URL が発行される。Resend 経由でメール送付も自動。
  4. 招待された人が URL を開く指定メールで OTP ログイン → /profile 入力 → 役割が即時付与 → /admin/secrets へ転送
📋 URL を手動で送りたい場合

Resend が落ちていてもフォーム送信後に「招待リンクを発行しました」と URL が表示されます。LINE / Slack / Discord などで直接共有可能。

権限の剥奪

/admin/team のメンバー一覧で各行の「権限剥奪」ボタンで member_type を member に戻す。owner (mail@yukihamada.jp) は剥奪不可（システムで保護）。

LINE Login のセットアップ

LINE Login を有効にすると、「LINE で続行」ボタンがゲートページと /profile に追加され、OTP メールなしでログイン可能になります。

  1. LINE Developers Console で Login channel を作成https://developers.line.biz/console/ → 新規 Provider → Login channel
  2. Callback URL に https://solun.art/api/soluna/auth/line/callback を登録
  3. scope を profile openid email に設定email は LINE で電話認証済みのユーザーから取得可
  4. Channel ID + Channel Secret を Fly secrets にセットfly secrets set LINE_LOGIN_CHANNEL_ID=… LINE_LOGIN_CHANNEL_SECRET=… -a soluna-web
  5. deploy 後、auto-detection でボタン表示/api/soluna/auth/line/available が enabled:true を返したら OK

SOLUNA 既存メールと LINE アカウントの紐付け

  • LINE で初回ログインした際、LINE 連携メールが SOLUNA に既存 → そのアカウントに line_user_id を紐付け
  • メールが新規 → 新規 soluna_members 行を作成（LINE 表示名を name にプリフィル）
  • LINE が email を返さなかった場合 → 内部用 sentinel email line:<id>@line.local で作成
⚠ LINE のみで作ったアカウントは admin になれない

LINE Login で新規作成された行は member_type=member。 admin にするには owner が /admin/team から該当メールに対し「招待」を発行 → 受諾フローを通す。

08. KAGI iOS app による物理ロック自動同期

SOLUNA の DB を変えると、KAGI iOS app（管理者の iPhone）が自動で SwitchBot Lock に書き込みを行います。
これにより「DB は新しいが現地ロックが古いまま」という事故を防ぎます。

仕組み（概要）

/admin/secrets で⟳ ローテート
    ↓ DB 更新 (door_code = 530955)
    ↓ lock_sync_status = 'pending'
KAGI iOS app (yuki iPhone)
    ↓ 15 分ごとに poll /api/soluna/owner/property-secrets/<slug>
    ↓ home.doorCode != 新コード → 差分検知
    ↓ SwitchBotClient.createPermanentPassword(...)
    ↓ POST sync-ack {status: 'ok', key_id: ...}
soluna-web: lock_sync_status = 'ok'
   ↓ /admin/secrets バッジ 🟢 Lock synced

初回セットアップ手順（物件 1 つにつき 1 回）

  1. SwitchBot Lock を物件玄関に設置物理取付け + ペアリング。SwitchBot 公式アプリで一度動作確認
  2. SwitchBot OpenAPI トークン / Secret を取得SwitchBot 公式アプリ → プロフィール → 開発者向け → トークン取得
  3. KAGI iOS を開き「物件を追加」SOLUNA portfolio から自動取り込み or 手動で名前入力
  4. 該当 Home の設定画面で以下を入力SwitchBot トークン / Secret / Lock デバイス ID（SwitchBotClient.fetchDevices で一覧化）
  5. 「🔄 鍵を同期」ボタンで初回テストSOLUNA の現在 door_code が SwitchBot に書き込まれる。成功すると Toast「鍵を同期しました」
  6. /admin/secrets で 🟢 Lock synced を確認

運用

  • KAGI iOS app は foreground 復帰時と 15 分の Background Refresh で自動 sync
  • iPhone がスリープ / 機内モードでも、復帰後に自動再開
  • 手動同期: 各 Home パネルの「🔄 鍵を同期」ボタン
🚨 大事な制約

KAGI iOS app は 管理者の iPhone 1 台で全 4 物件の同期を担います。
その iPhone を紛失 / 機種変したら、新端末で再セットアップが必要。
バックアップとして iCloud Drive / Keychain Backup を有効化推奨。

トラブル: SwitchBot に書き込まれない

症状原因対処
🟡 Syncing が 15 分以上続くKAGI iOS が Background Refresh 停止アプリを起動
🔴 Lock sync failed (battery low)SwitchBot のバッテリー切れ電池交換 → 「🔄 鍵を同期」
🔴 Lock sync failed (offline)SwitchBot Hub が WiFi 切断現地ルーター再起動
🔴 Lock sync failed (auth)SwitchBot トークン期限切れ新トークン取得して KAGI Home に再入力
🔴 Lock sync failed (invalid passcode)door_code が SwitchBot 制約違反6 桁数字に修正

09. 緊急時の対応フロー

火事 / 救急（人命優先）

  1. 119 へ通報（消防・救急）場所を伝える。物件名（例: SOLUNA 美留和 THE LODGE）+ 住所
  2. ゲストを屋外に避難無理な救助は不要。ゲストとスタッフの命を優先

10. 監査ログ / トラブルシュート / ヘルスチェック

変更履歴を画面で確認 — /admin/audit

SOLUNA admin で /admin/audit を開くと、全 4 物件のシークレット変更を時系列でブラウズできます。物件・action・期間・件数でフィルタ可能。

UI 項目意味
📊 統計バー上部期間内の rotate / edit 件数、直近の変更者
action chiprotate = 鍵ローテ、edit = フィールド編集、manual_set = API 直接更新
old → new変更前後の値。null は ∅ で表示
by変更者 email + 接続元 IP

監査ログは 90 日保持。それ以降は自動削除（6 時間ごとの cleanup）。

サーバの状態を一望 — /api/soluna/admin/health

運用ダッシュボードや Telegram bot から叩いて、サーバの健康状態を 1 リクエストで確認できます。

curl https://solun.art/api/soluna/admin/health \
  -H "Authorization: Bearer $TOKEN" \
  -H "X-Requested-With: XMLHttpRequest"

レスポンスに含まれる主要メトリクス:

  • audit_events_last_24h — 過去 24 時間の変更件数
  • properties_pending_sync — KAGI iOS が SwitchBot 書き込み待ちの物件数
  • properties_with_sync_error — 失敗中の物件数
  • total_in_flight_guests — 現在ご滞在中のゲスト総数（rotate 安全判定に使用）
  • pending_admin_invites — 未受諾の admin 招待数
  • properties.*.kagi_device_registered — 各物件で SwitchBot device id がセットされているか

毎朝の自動 smoke チェック

GitHub Actions が毎朝 09:00 JST に scripts/smoke-public.sh を本番に対して実行します。 32 項目（認証ゲート / CSRF 防御 / slug ホワイトリスト / カノニカル URL リダイレクト / セキュリティヘッダ / OTP 応答形式）の regression を 1 つでも検知すると、Telegram @yukihamada_ai_bot に 🔴 SOLUNA smoke-public FAILED 通知が飛びます。

よくあるトラブル

マニュアルページが 404 になる
slug 名が誤り。atami / lodge / nesting / instant のいずれか。新物件は MANUAL_SLUGS に追加が必要。
ローテーション後にゲストから「鍵が開かない」と連絡
1) /admin/secrets を開いて新鍵番号を再確認 → 2) ゲストに新番号をメール/電話で送る → 3) SwitchBot Lock が物理的に書き換わっているか「🔄 鍵を同期」ボタンで確認。
/admin/secrets で「未設定」が消えない
該当フィールドの ✎ から実値を入力して保存。Wi-Fi PW は物理ルーターと一致確認後に。
CSRF エラー「missing X-Requested-With」
curl で直接叩いた場合に発生。-H "X-Requested-With: XMLHttpRequest" を追加。
Rate limit エラー (HTTP 429)
1 分に 20 回以上の PUT or 10 回以上の rotate でブロック。1 分待って再試行。

11. よくある質問 (FAQ)

Q1. 鍵番号はどのくらいの頻度でローテートすべき？
最低でも 30 日に 1 回、理想は チェックアウトごと。次のゲストの安全のため。
Q2. 鍵番号 6 桁は推測されないの？
6 桁 = 100 万通り。スマートロックの試行回数制限と組み合わせれば実質安全。旧コードは物理ロック書き換え時点で無効化され、DB 上の previous_door_code 列に 7 日間「監査用」に残るのみ。
Q3. KAGI iOS app を入れている iPhone を機種変したら？
新端末で KAGI を再インストール → Apple ID 同期 or iCloud Drive backup から復元 → SwitchBot トークンの再入力（必要なら）。
Q4. 複数の管理者が同時に編集したら？
現在は last-write-wins。編集前に他管理者と口頭調整を。今後 optimistic lock を実装予定。
Q5. ゲストにマニュアル URL を案内するタイミングは？
予約確定メールに https://solun.art/<slug>/manual を記載。ゲストは OTP ログイン後に閲覧可能。
Q6. ゲストから「マニュアルが見れない」と連絡
予約日が check-in 14 日前 〜 check-out 7 日後の窓に入っているか確認。窓外はアクセス不可。
Q7. 新物件を追加するには？
server.js の MANUAL_SLUGS / MANUAL_PROPERTY_NAME / PROPERTY_SECRET_DEFAULTS に追加 + cabin/<slug>-manual.html を作成 + /admin/secrets が自動表示。
Q8. Wi-Fi パスワードは KAGI iOS 経由で同期される？
door_code と同じく KAGI iOS が poll → home.wifiPassword を更新。ただし物理ルーターは別途手動更新が必要（KAGI app からの API 書き込み不可）。
